# Scratch 项目审阅报告

## 总体评价

Scratch 是一个高质量的跨平台 Markdown 笔记应用，技术栈现代（Tauri v2 + React 19 + TipTap v3 + Tantivy 0.22 + Tailwind v4），代码质量整体优良，架构设计合理，性能优化到位。以下是详细分析。

---

## 一、优点

### 1. 技术栈选型优秀
- 所有依赖版本对齐且为最新稳定版，Tauri v2 生态完整
- React 19 + TipTap v3 + Tailwind v4 + Vite 7 = 最前沿的技术栈
- 版本号在 `package.json`、`Cargo.toml`、`tauri.conf.json` 三处同步（0.10.0）

### 2. 前端架构设计精良
- **双 Context 模式**：NotesContext 将数据和 actions 分离，避免不必要的 re-render
- **三层 stale-request 防护**：通过单调递增计数器防止竞态条件
- **防抖全面**：auto-save 500ms、搜索 150-220ms、文件监听 300ms、git 刷新 1000ms
- **Ref 驱动的脏标记**：`needsSaveRef` 避免每次按键都计算 markdown
- **React.memo + useCallback/useMemo** 使用到位（173+ 处 useCallback/useMemo）
- **Service 层抽象**：所有 Tauri invoke() 调用封装在 `src/services/` 中

### 3. Rust 后端安全可靠
- **路径安全**：`abs_path_from_id` 防止路径穿越，`validate_folder_path` 校验目录路径
- **URL 安全**：`open_url_safe` 仅允许 http/https/mailto 协议
- **平台感知**：Windows/macOS/Linux 各有适配，`#[cfg]` 守卫正确
- **async 运行时不阻塞**：CPU 密集操作使用 `spawn_blocking`
- **unsafe 代码仅一处**：Windows DWM 标题栏着色，范围可控

### 4. 构建与 CI/CD 完善
- CI 验证前端构建 + Rust 编译 + clippy（warnings 视为 error）
- Release 支持三平台（macOS universal binary + Ubuntu + Windows）
- macOS 代码签名完整，auto-updater 启用

### 5. 安全措施到位
- CSP 严格（self + ipc + asset），仅样式使用 unsafe-inline
- Tauri capabilities 权限最小化
- 文件关联（.md/.markdown）正确配置

### 6. TypeScript 质量极高
- 全量 TypeScript 严格模式，无 `@ts-ignore`
- 仅 2 处 `any`（均为 TipTap 类型系统限制，已做最终类型断言）
- 零 TODO/FIXME/HACK

## 二、问题与改进建议

### 🔴 高优先级

#### 1. lib.rs 过于庞大（3965 行）
**位置**：`src-tauri/src/lib.rs`

所有 Tauri 命令（45 个）、数据类型、搜索索引、文件监听、AI 集成、预览窗口、应用初始化全在一个文件中。

**建议拆分**：
```
src-tauri/src/
  lib.rs          -> 仅保留 run() 和 setup
  commands/
    notes.rs      -> 笔记 CRUD
    folders.rs    -> 文件夹管理
    settings.rs   -> 设置管理
    search.rs     -> 搜索命令
    file_io.rs    -> 文件读写
    clipboard.rs  -> 剪贴板操作
    git.rs        -> Git 命令（从 git.rs 迁移）
    ai.rs         -> AI CLI 集成
    preview.rs    -> 预览窗口
  models.rs       -> 数据结构
  state.rs        -> AppState 和配置
  search.rs       -> Tantivy 索引
  watcher.rs      -> 文件监听
```

#### 2. 零测试覆盖
项目没有任何单元测试或集成测试（Rust 和 TypeScript 均为零）。

**建议**：
- Rust：至少为路径安全、markdown 解析、搜索索引编写单元测试
- TypeScript：为 `src/lib/utils.ts`、`src/lib/plainText.ts` 等纯函数编写测试
- 考虑添加 Vitest 作为前端测试框架
- 考虑 Rust 集成测试覆盖关键 Tauri 命令

#### 3. 正则表达式重复编译
**位置**：`src-tauri/src/lib.rs:551-593`（`strip_markdown` 函数）

每次调用编译 4 个正则表达式，而此函数在 `list_notes`（遍历所有笔记时调用）和搜索结果中被频繁调用。

**建议**：使用 `std::sync::OnceLock` 或 `lazy_static!` 将正则编译为静态量。

### 🟡 中优先级

#### 4. Editor.tsx 过大（2618 行）
**位置**：`src/components/editor/Editor.tsx`

承担了太多职责：TipTap 配置、搜索、链接/数学弹窗、源码模式、复制/导出、图片粘贴、表格右键菜单等。

**建议拆分**：
- `EditorSearch.tsx` - 笔记内搜索逻辑
- `FormatBar.tsx` - 格式化工具栏（当前内联定义，约 184 行）
- `SourceMode.tsx` - 源码模式逻辑
- `EditorPopups.tsx` - 链接/数学弹窗管理

#### 5. `let _ =` 静默错误抑制
**位置**：`src-tauri/src/lib.rs`（约 49 处）

部分静默丢弃错误是合理的（发送事件、聚焦窗口），但以下场景值得关注：
- 搜索索引更新失败被静默吞掉（line 860, 1121-1123）
- 设置保存失败被静默吞掉（line 1501, 1599, 1704）

**建议**：对搜索索引和设置保存的错误添加 `tracing::warn!` 日志。

#### 6. Mutex `.expect()` 使用（87 处）
所有 `RwLock`/`Mutex` 的 lock 调用都使用 `.expect()`。如果锁被 poison，整个应用会 panic。

**建议**：对于非关键路径，考虑使用 `try_read()`/`try_lock()` + 降级处理。或者至少使用 `tracing::error!` 记录 poison 事件。

#### 7. `list_notes` 性能
**位置**：`src-tauri/src/lib.rs:889`

每次调用都通过 `WalkDir` 遍历整个目录树，并读取每个 `.md` 文件内容提取标题和预览。对于大量笔记的 vault 这是性能瓶颈。

**建议**：
- 利用文件 mtime 与缓存对比，跳过未修改的文件
- 搜索时使用 Tantivy 索引而非全量遍历
- 考虑增量缓存策略

#### 8. Tantivy 索引操作非批量提交
**位置**：`src-tauri/src/lib.rs:224, 232`

每次 `index_note`/`delete_note` 后都执行 `commit()`，在文件监听器处理批量事件时会造成 I/O 尖峰。

**建议**：实现批量提交（buffer + debounce commit），或在 `rebuild_index` 中使用现有的批量 commit 模式。

#### 9. Windows 下 `get_expanded_path` 局限
**位置**：`src-tauri/src/lib.rs:2767-2810`

该函数使用 `HOME` 环境变量和 Unix 风格路径（`~/.nvm`、`~/.volta/bin`、`/usr/local/bin`），在 Windows 上 `HOME` 通常未设置（使用 `USERPROFILE`），可能导致 CLI 工具找不到。

**建议**：添加 Windows 特定路径支持（`%APPDATA%`、`%LOCALAPPDATA%` 等）。

### 🟢 低优先级

#### 10. CustomEvent 类型不安全
多个组件通过 `window.dispatchEvent(new CustomEvent(...))` 通信，事件类型是隐式的（非类型化）。

**建议**：定义类型化的事件映射接口，或改用 Context 传递。

#### 11. Settings 重复加载
`NoteList.tsx`、`Editor.tsx`、`CommandPalette.tsx`、`Sidebar.tsx` 各自独立调用 `getSettings()`。

**建议**：考虑通过 Context 共享 settings，减少重复的 IPC 调用。

#### 12. 无 ESLint 配置
项目依赖 TypeScript 严格模式但没有 ESLint。

**建议**：添加 ESLint + Prettier 配置以统一代码风格（虽然当前代码质量已经很高）。

#### 13. 文件监听器批量事件处理
**位置**：`src-tauri/src/lib.rs:2185-2215`

文件监听器对每个事件单独加锁和处理，批量文件变动时（如 git checkout）可能效率不高。

**建议**：实现事件聚合（coalescing），将短时间窗口内的多个事件合并处理。

## 五、总结

Scratch 是一个架构清晰、代码质量优秀、性能优化到位的现代化笔记应用。主要改进空间在于：
1. **测试覆盖**（当前为零）
2. **后端模块化**（lib.rs 单文件过大）
3. **前端核心组件拆分**（Editor.tsx）
4. **少量性能优化**（正则缓存、批量索引、增量缓存）

整体而言，这是一个 **B+** 级别的项目，技术选型前沿，代码规范良好，安全意识强，主要短板在测试和模块化上。
